What Programming Languages should you learn as a CS student and why

Which programming language should I learn first? This is the most obvious question which any student would probably ask himself the minute he/she is going to sign-up for a computing course. Many computer science courses start with JavaScript or Python and that’s because of its simplicity and easy syntax, but it is not said that this is going to work out for you or not in the long run (Tozzi, 2018). 

It is however true, that no matter where you want your computer science education to lead you, you must grasp first some basic knowledge of what a programming language is, its syntax, what can handle, what is does and what you can achieve with it and then move onto complex things. 

Speaking from my personal experience, I believe that the minute you start a computing course or a bootcamp, many students get it in their heads that the most important thing is to “learn how to code”, no matter what programming language is taught. After learning the basics, they keep diving into it, eventually start looking at other programming languages but fail to stop and think for a minute about what I want to learn “to do with that” or are there any benefits of that programming language in the long run. Is this going to help me to secure a graduate or a junior role the minute I am going to graduate ?

As a student or not limited to, if you want to start with straightforward language because you want to grasp quickly the fundamental concepts of programming before going and learning a high level, Object Orientated Programming (OOP) language like Java, that’s great.

For some students, JavaScript or HTML will be the best choice in terms of programming language to learn as their first. Being used for the development of the front-end web application, so it’s a “must-have” skills to put on your resume for those wanting to work in these fields. However, learning a language like JavaScript, CSS and HTML won’t help you much if you seek employment with a mobile development development company with primarily focus is on native iOS and Android app development. Same goes for enterprise applications, where big companies use Java as the preferred backend programming language. According to StackShare, nearly 90.000 companies reportedly use Java in their tech stacks, including Google, Facebook, Airbnb, Uber,  Netflix, Bank of Ireland, just to name a few (Stackshare, 2019). 

With that in mind it is very important to keep an open mind when choosing the best coding language to learn. Starting a career in computer science isn’t as easy as graduating and securing a job interview. Even though, each year there is a high demand for new graduates, the candidate’s competitive advantage comes from specific expertise acquired during his years in college as well as the programming languages he studied or self taught up to that point. Once you set your priorities right and have the proper skills under your belt the success is guaranteed. 

How do Blockchains work?

What is a blockchain?

Blockchain is the technology that sits behind cryptocurrencies such as BitCoin, Ethereum, Ripple just, to name a few, but it is not limited to them. It can be programmed to track anything: medical records, bank transactions and many various things with sensitive information. Just as the name itself suggests, it is a chain of blocks, and each of these boxes contain information. 


There can be different types of blockchains, depending on the type of data being stored.  The primary types of blockchains are: Public blockchains like Bitcoin and Ethereum, Private blockchains like Hyperledger and R3 Corda and Hybrid blockchains like Dragonchain.

Public meaning that the blockchain is open source and therefore anyone is allowed to participate as miners, developers or community members. On the other hand the private blockchain, as the name suggests it is not open to anyone, therefore participants need access to join the network, while the transactions taking place are private and available only to those that have been authorized to join the network. Lastly, the hybrid blockchain incorporate both benefits of the private and public blockchain. This gives users the option to cherry pick what data they want to make public and what data they want to keep private.

But why Blockchain?

Is Blockchain secure? Once the information has been registered within a blockchain, it cannot be changed anymore. That is because, each block contains its own hash and the hash of the previous block. A ‘hash’ can be seen just as a fingerprint, in other words, it is a key.  

One characteristic of the hash is that it’s always unique. If you were to change something inside the block, the hash would change automatically.

So, if the hash of a block is changed, it automatically fails validation.

Although, using hashes by itself obviously it would not be enough to prevent tampering. There would be software that could calculate hundred thousands of hashes per second. In order to avoid this, blockchains also use a ‘proof-of-work’ function. 

What the proof-of-work function’s role is to slow down the creation of new blocks. For example, in BitCoin’s case it takes about 10 minutes to calculate the required proof-of-work and add a new block to the chain. This mechanism makes it hard for any tampering to be done. If any tampering would be attempted, the proof-of-work would need to be re-calculated as well.

Another way that blockchains secure themselves is that they are being distributed. What this means is that it uses P2P ( Peer to Peer) Network and anyone is allowed to join. When somebody joins this network, they get a copy of the blockchain. After somebody creates a new block, every single person in the network receives the same block. Then the blocks are being verified for validation purposes. If everything seems to be in order, the nodes then add the block to the blockchain. All the nodes from a blockchain communicate and agree with one another if a block is valid or not.

Regardless of what the future holds, blockchain technology is growing massively because almost anything in the world of today involves online transactions which can work in a blockchain system. It is just a matter of time before blockchain technology will change how we do business interact with one another.
